# Competitive Solutions HR

نسخة إنتاجية أولية لنظام موارد بشرية خاص بشركة الحلول التنافسية، مبنية باستخدام `Flask` و`SQLite`.

## المزايا

- تسجيل دخول فعلي بجلسات مستخدم.
- صلاحيات منفصلة بين `admin` و`employee`.
- إدارة الحضور والانصراف مع تصدير CSV.
- إدارة مسيرات الرواتب مع تنزيل مسير الراتب.
- قاعدة بيانات SQLite مع بيانات تجريبية جاهزة.

## التشغيل المحلي

```bash
python -m pip install -r requirements.txt
python app.py
```

ثم افتح:

```text
http://127.0.0.1:5000
```

## الحسابات الافتراضية

- الإدمن: `admin@competitive.local` / `Admin@123`
- الموظف: `ahmed@competitive.local` / `Employee@123`

يفضل تغيير كلمات المرور قبل الاستخدام الفعلي.

## قاعدة البيانات

- المسار الافتراضي المحلي في هذه النسخة: مجلد مؤقت النظام.
- يمكن تخصيص مسار البيانات عبر متغير البيئة `CSCO_DATA_DIR`.

## النشر على Render

هذا المشروع يحتوي على ملف [render.yaml](./render.yaml) جاهز.

خطوات النشر:

1. ارفع المشروع إلى GitHub.
2. أنشئ خدمة جديدة في Render من المستودع.
3. Render سيقرأ `render.yaml` تلقائيًا.
4. أضف متغيرات البيئة عند الحاجة:
   - `SECRET_KEY`
   - `CSCO_DATA_DIR` إذا أردت تخصيص مكان قاعدة البيانات

## ملاحظة إنتاجية

`SQLite` مناسبة كبداية سريعة أو لعدد مستخدمين محدود. عند التوسع الفعلي يفضل نقل التخزين إلى PostgreSQL وربط النظام بخدمة ملفات لكشوف الرواتب والمرفقات.
